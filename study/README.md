# Teacher Construct Importance Study

A jsPsych-based forced-choice experiment for understanding which student qualities K-12 teachers consider most important to assess.

**Author**: Arno Klein (arnoklein.info)  
**License**: Apache v2.0

---

## Overview

This study presents K-12 teachers with pairs of student qualities (e.g., "Self-Control" vs. "Empathy") and asks them to choose which is more relevant and important for them to understand when reflecting on their students. The data are used to estimate a ranked importance hierarchy across 90 candidate constructs, identify constructs that are perceived as interchangeable, and validate construct labels via synonym equivalence.

---

## Research Questions

1. **Importance hierarchy**: Which constructs do teachers prioritize when reflecting on their students?
2. **Construct dependence**: Which constructs are perceived as interchangeable vs. distinct?
3. **Construct validity**: Do target items and their synonyms show equivalent importance patterns?

---

## Stimuli

- **90 target items**: Candidate constructs (student qualities), e.g. "Self-Control", "Empathy"
- **2 synonyms per item**: Alternative labels for the same construct, e.g. "Emotional regulation", "Compassion"
- Source file: `data/items.csv` (columns: `Items`, `Synonym 1`, `Synonym 2`)

Each construct is represented in a given participant's session by **either** the canonical item name **or** one randomly selected synonym — never both. This means the pairwise comparisons involve canonical items and/or synonyms, but never two labels for the same construct.

---

## Study Design

### Per-participant assignment

Each participant is assigned **K = 26** constructs, randomly drawn from the full set of 90 using a balanced incomplete block design pre-generated by `generate_subsets.py`. Assignment is deterministic from the participant's Prolific ID (PID hash → subset index), so it is reproducible and requires no server-side session management.

For each of the 26 assigned constructs, either the canonical item or one of its synonyms is chosen at random as the displayed term.

### Trials

Each participant completes all **C(26, 2) = 325** pairwise comparisons within their assigned subset — a complete within-subset design. Pairs are presented in random order.

**Prompt** (shown above every pair):
> *"When considering your students, which of the two presented qualities is more relevant and important for you to understand or assess? Please use your professional judgment."*

**Response**: Forced binary choice (click one of two buttons).

**Estimated duration**: ~19 minutes at 3.5 s/trial.

### Attention checks

One attention check is inserted approximately every 55 trials (~5 min). Each check presents a nonsense pair with an obvious correct answer (e.g., "Paying attention" vs. "Continuing to stop") and instructs the participant to select a specific word. Failing any attention check immediately ends the study and routes the participant to a separate Prolific completion code.

### Background questions

Before the main task, participants answer six background questions:
- Grade levels currently taught (multi-select: kindergarten, primary, middle, high school, college, vocational)
- Whether they teach students with special needs (yes/no)
- School type (public/private)
- Years of teaching experience (1 / 2–5 / 6–10 / >10)
- Average class size (<15 / 15–25 / >25)
- Country of work (dropdown)

### Sample

- **Target N**: ~300 K-12 teachers with English as primary language, recruited via Prolific
- **Subset coverage**: With 400 pre-generated balanced subsets, each construct appears in ~115 sessions and each cross-construct pair is observed by ~32 independent judges on average
- **Cost estimate**: ~$12/hr × ~0.4 hr × 300 = ~$1,440 + Prolific fees

## Analysis Plan

### 1. Importance ranking — Bradley-Terry model

The Bradley-Terry (BT) model estimates a latent "worth" parameter for each of the 90 constructs from the pairwise choice data. Because each participant completes a complete within-subset design (all C(26,2) pairs for their 26 assigned constructs), the comparison graph across participants is well-connected, enabling stable BT estimation even with an incomplete cross-participant design.

**Output**: Ranked list of 90 constructs by teacher-perceived importance, with confidence intervals on BT worth parameters.

Key fields used: `chosen_construct_id`, `unchosen_construct_id`, `subset_index`.

### 2. Construct dependence — residual covariance clustering

After fitting the BT model, examine the matrix of residuals (observed − expected win probabilities). Constructs that the model consistently fails to order (high residual covariance between two constructs' opponent profiles) are candidates for measuring overlapping or interchangeable concepts.

**Approach**: Cluster constructs by their residual covariance profiles (hierarchical clustering or spectral methods). Items with high within-cluster residual covariance may warrant merging or further disambiguation.

**Output**: Dependence matrix; candidate construct clusters.

### 3. Construct validity — synonym win-rate correlation

Because each construct is represented by either its canonical label or a synonym across different participants, we can compare the construct's performance profile under each label.

For each canonical item with at least one synonym tested:
- Collect the win-rate vector of the canonical term against all opponents it faced (across all participants assigned that term)
- Collect the win-rate vector of the synonym against all opponents it faced
- Compute the correlation between the two vectors

**Interpretation**:
- **High correlation** → the synonym captures the same construct (good validity)
- **Low correlation** → the synonym evokes a different concept (poor validity; consider relabeling)

Additionally, a likelihood ratio test can compare a BT model that constrains a canonical item and its synonym to equal worth parameters against an unconstrained model.

**Output**: Validity coefficient (Pearson r of win-rate vectors) per canonical–synonym pair; LRT p-values.

Key fields used: `chosen_canonical`, `chosen_is_synonym`, `unchosen_canonical`, `unchosen_is_synonym`.

## Data Output

One CSV file per participant, uploaded to OSF and saved to `localStorage` as a fallback.

### CSV columns

| Column | Description |
|---|---|
| `user_id` | Prolific participant ID |
| `task` | `demographics`, `forced_choice`, or `attention_check` |
| `trial_index` | Position within forced-choice sequence (0–324) |
| `subset_index` | Which of the 400 pre-generated subsets was assigned |
| `left_term` / `right_term` | Displayed terms (canonical or synonym) |
| `left_construct_id` / `right_construct_id` | Integer index (0–89) of the underlying construct |
| `left_canonical` / `right_canonical` | Canonical item name for each construct |
| `left_is_synonym` / `right_is_synonym` | Whether the displayed term is a synonym |
| `chosen_*` / `unchosen_*` | Same fields for the chosen and unchosen options |
| `response_time` | Time to respond in milliseconds |
| `correct_word` / `chosen_word` / `passed` | Attention check fields (attention_check rows only) |
| `grade_levels` / `special_needs` / `school_type` / `experience` / `class_size` / `country` | Background fields (demographics row only) |

## Configuration

Edit `src/experiment.js` to change design parameters:

```javascript
let experimentConfig = {
  termsPerParticipant: 26,           // K; determines C(K,2) pairs per participant
  subsetsFile: 'data/subsets.json',  // Pre-generated subset assignments
  screensBetweenAttentionChecks: 55, // ~1 check per 5 min at 3.5s/trial
};
```

Regenerate subsets whenever `termsPerParticipant` or `data/items.csv` changes:

```bash
python3 generate_subsets.py --k 26 --n 400
```
